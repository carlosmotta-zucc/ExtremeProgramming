1. ESTORIAS DE USUARIO (Leitura)
----------------------------------------------------------------

US01 - Cadastrar Aluno
   "Como atendente, quero cadastrar um aluno para manter seus
    dados no sistema."
   Criterios de Aceitacao:
     - O sistema deve permitir informar nome e e-mail do aluno.
     - O aluno deve ficar armazenado no sistema.
     - Nao deve ser possivel cadastrar aluno com nome vazio.

US02 - Cadastrar Curso
   "Como atendente, quero cadastrar um curso para disponibiliza-lo
    para matricula."
   Criterios de Aceitacao:
     - O sistema deve permitir informar nome do curso e quantidade
       de vagas.
     - O curso deve ficar armazenado no sistema.
     - A quantidade de vagas deve ser maior que zero.

US03 - Matricular Aluno em Curso
   "Como atendente, quero matricular um aluno em um curso para
    registrar sua inscricao."
   Criterios de Aceitacao:
     - So e possivel matricular um aluno ja cadastrado.
     - So e possivel matricular em curso ja cadastrado.
     - A matricula deve ficar registrada no sistema.
     - Nao deve ser possivel matricular se o curso ja estiver
       com todas as vagas ocupadas. O sistema deve informar
       o nome do curso e a quantidade de vagas/matriculados.

US04 - Listar Dados do Sistema
   "Como atendente, quero listar alunos, cursos e os alunos
    matriculados em um curso especifico para visualizar o estado
    atual do sistema."
   Criterios de Aceitacao:
     - O sistema deve permitir listar TODOS os alunos cadastrados.
     - O sistema deve permitir listar TODOS os cursos cadastrados.
     - Dado um ID de curso, o sistema deve listar todos os alunos
       matriculados nesse curso.
     - Se o ID de curso informado nao existir, o sistema deve
       informar erro claramente.
     - Listas vazias devem ser informadas ao usuario (sem erro).

3. ESTIMATIVAS DE COMPLEXIDADE
----------------------------------------------------------------

Escala adotada (Story Points - sequencia simplificada):
   1 = Muito simples    2 = Simples    3 = Media
   5 = Alta             8 = Muito alta

+--------+------------------------------+--------+----------+
|  ID    |  Estoria                     | Pontos | Esforco  |
+--------+------------------------------+--------+----------+
| US01   | Cadastrar Aluno              |   2    | Simples  |
| US02   | Cadastrar Curso              |   2    | Simples  |
| US03   | Matricular Aluno em Curso    |   3    | Media    |
| US04   | Listar Dados do Sistema      |   2    | Simples  |
+--------+------------------------------+--------+----------+
| TOTAL DA ITERACAO                     |   9    |          |
+---------------------------------------+--------+----------+

Justificativa das estimativas:
   - US01 (2 pts): cadastro basico com 1 validacao (nome vazio).
   - US02 (2 pts): cadastro basico com 1 validacao (vagas > 0).
     Mesmo nivel de US01.
   - US03 (3 pts): depende de US01 e US02; envolve duas
     verificacoes de existencia e criacao de relacionamento
     entre entidades.
   - US04 (2 pts): consulta sobre estruturas ja existentes
     (listas de alunos, cursos e matriculas); inclui 1 caso
     de validacao (curso inexistente) e 1 submenu no console.

4. ESTORIAS ESCOLHIDAS PARA A PRIMEIRA ITERACAO
----------------------------------------------------------------

Selecionadas para a iteracao 1 (1 semana):

   [X] US01 - Cadastrar Aluno
   [X] US02 - Cadastrar Curso
   [X] US03 - Matricular Aluno em Curso
   [X] US04 - Listar Dados do Sistema

Total estimado: 9 pontos

Ordem de implementacao (com base na dependencia):
   1) US01 - Cadastrar Aluno   (pre-requisito de US03 e US04)
   2) US02 - Cadastrar Curso   (pre-requisito de US03 e US04)
   3) US03 - Matricular Aluno em Curso
   4) US04 - Listar Dados do Sistema (depende de US01..US03)

Motivacao da priorizacao (cliente):
   US01..US03 entregam o fluxo minimo de valor (sem aluno e
   sem curso nao ha matricula). US04 foi incorporada na mesma
   iteracao apos pedido do cliente: sem listagem nao e possivel
   conferir visualmente o que foi cadastrado nem auditar quem
   esta matriculado em cada curso. Mantem o principio XP de
   "fazer o mais simples que funcione" e habilita a demo
   completa ao final da iteracao.
